# Frontend Scaffolding Context

## CRITICAL: Single File Architecture

**ALL frontend code MUST be in ONE file: `App.jsx`**

The LLM will generate a complete, self-contained React application in App.jsx (600-900 lines).
This file will be COMPLETELY REPLACED by the generated code.

## File Structure
```
frontend/src/
├── main.jsx    # Entry point - DO NOT MODIFY (provides BrowserRouter)
├── App.jsx     # ALL CODE GOES HERE - generated by LLM
└── App.css     # Tailwind imports (optional CSS additions)
```

## App.jsx Required Sections (in order)
1. **Imports** - React, react-router-dom, axios, toast, heroicons, date-fns, clsx
2. **API Client** - axios instance with auth interceptor + all API functions
3. **AuthContext & AuthProvider** - Complete auth state management
4. **useAuth hook** - Access auth context
5. **LoadingSpinner** - Reusable loading component
6. **ProtectedRoute** - Auth guard for protected pages
7. **Navigation** - Header with nav links
8. **HomePage** - Public landing page with guest/authenticated views
9. **LoginPage** - Login form with validation and error handling
10. **RegisterPage** - Registration form with password confirmation
11. **UserPage** - Full CRUD interface for main entity
12. **AdminPage** - Dashboard with stats, data table, bulk actions
13. **App** - Main component with Routes (NO BrowserRouter - main.jsx provides it)
14. **export default App**

## Available Packages (DO NOT ADD OTHERS)
```
react, react-dom, react-router-dom, axios, react-hot-toast, @heroicons/react, date-fns, clsx, uuid
```

## Code Quality Requirements
- NO placeholders, NO TODOs, NO incomplete functions
- Proper error handling with try/catch
- Loading states for all async operations
- Toast notifications for success/error feedback
- Empty states when no data
- Form validation
- Responsive Tailwind CSS styling
