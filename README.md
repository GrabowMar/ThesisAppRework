# ThesisAppRework - AI Model Analysis Platform

A Flask-based web application for researching and testing AI-generated applications. The system provides comprehensive analysis capabilities including security scanning, performance testing, and batch processing for AI model-generated web applications.

## 🎯 Project Overview

This platform analyzes and tests web applications generated by various AI models (GPT, Claude, Gemini, etc.) to evaluate their quality, security, and performance characteristics. It features:

- **Web Interface**: HTMX-powered dashboard for managing analyses
- **Containerized Testing**: Docker-based isolation for testing generated applications
- **Security Analysis**: Automated security scanning with multiple tools
- **Performance Testing**: Load testing and performance benchmarking
- **Batch Processing**: Celery-based async processing for large-scale analysis
- **Real-time Monitoring**: Live updates and progress tracking

## 🏗️ Architecture

### Core Components

```
src/app/
├── routes/          # Flask blueprints for web interface
├── services/        # Business logic and external integrations
├── models.py        # SQLAlchemy database models
├── tasks.py         # Celery background tasks
├── templates/       # Jinja2 templates with HTMX
└── static/         # CSS, JavaScript assets
```

### Service Architecture

- **Web Layer**: Flask blueprints with HTMX for dynamic UI
- **Service Layer**: Modular services for different functionalities
- **Task Queue**: Celery with Redis for async processing
- **Database**: SQLite (dev) / PostgreSQL (prod) with SQLAlchemy
- **Containers**: Docker for isolating generated applications

## 🚀 Quick Start

### Prerequisites

- Python 3.11+
- Docker Desktop (for containerized testing)
- Git

### Development Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/GrabowMar/ThesisAppRework.git
   cd ThesisAppRework
   ```

2. **Create virtual environment**
   ```bash
   python -m venv .venv
   # Windows
   .venv\Scripts\activate
   # Linux/Mac
   source .venv/bin/activate
   ```

3. **Install dependencies**
   ```bash
   pip install -r src/requirements.txt
   ```

4. **Initialize database**
   ```bash
   cd src
   python -c "from app.extensions import init_db; init_db()"
   ```

5. **Start the application**
   ```bash
   python main.py
   ```

6. **Access the application**
   - Web Interface: http://localhost:5000
   - Health Check: http://localhost:5000/health

### Optional: Celery Workers

For background processing:

```bash
# Terminal 1: Start Celery worker
celery -A app.tasks worker --loglevel=info

# Terminal 2: Start Celery beat (scheduler)
celery -A app.tasks beat --loglevel=info
```

## 🔧 Configuration

### Environment Variables

```bash
# Flask Configuration
FLASK_ENV=development
SECRET_KEY=your-secret-key

# Database
DATABASE_URL=sqlite:///instance/thesis_app.db

# Celery (optional)
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0

# Docker (for containerized testing)
DOCKER_HOST=npipe:////./pipe/docker_engine  # Windows
```
